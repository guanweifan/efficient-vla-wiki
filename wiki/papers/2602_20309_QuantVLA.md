## Claim
- 页面定位：这是一篇 **training-free VLA PTQ / low-bit deployment** 论文；它的核心贡献是为 language backbone 与 DiT action head 设计量化校准机制，而不是提出新的 VLA 架构。
- QuantVLA 的核心命题是：VLA 的部署瓶颈不只来自模型规模本身，还来自 language backbone 与 diffusion-style action head 在低比特下的耦合失稳；因此，若想把 post-training quantization 真正用于 VLA，关键不是直接套用 LLM/VLM PTQ，而是显式校准 **attention temperature** 与 **output-head residual energy** 这两类在 DiT 动作头中最容易被量化破坏的尺度。
- 论文将 `QuantVLA` 定位为一个 **training-free**、**post-training** 的 VLA 量化框架，claim 自己是首个面向 VLA 的 PTQ 方法，也是首个成功量化 diffusion transformer action head 的方案。其主要设计包括 selective quantization layout、attention temperature matching (`ATM`) 和 output head balancing (`OHB`)。
- headline 结果需要拆开理解：
  - `70%` 对应的是量化模块上的相对内存节省；
  - “`exceeds full-precision baseline`” 对应的是特定 `LIBERO` 设置下的任务成功率结果；
  - “首个 VLA PTQ / 首个成功量化 DiT action head” 则是方法定位主张，不应和数值结果写成同一层结论。
- 更稳的主张是：**VLA-specific scale calibration 让 PTQ 可以安全进入 language + DiT action head 这一耦合区间**，而不是“所有 VLA 都能在任意低比特下稳定超过 FP16”。

## Methodology Index

- **VLA-specific PTQ framing**：把量化对象从单纯 LLM/VLM 扩展到 language backbone 与 DiT action head 的联合系统。
- **Selective quantization layout**：量化 language backbone 与 DiT 中的线性层，同时保留 attention projections 为 floating point，以避免破坏原始 operator schedule。
- **Attention Temperature Matching (ATM)**：通过 per-head scaling 对齐 teacher / quantized attention logits 的温度，抑制 quantization-induced softmax distortion。
- **Output Head Balancing (OHB)**：通过 per-layer residual interface calibration 修正 output projection 后的能量漂移。
- **Small unlabeled calibration buffer**：无需追加训练，只用小规模无标签校准缓冲区完成 scale calibration。
- **DiT fragility analysis**：论文先分析为什么 quantization scale drift 会在 diffusion-style action head 中放大失稳，再据此设计 ATM/OHB。

## Data Pointer

- **Abstract / introduction**：最清楚地给出 QuantVLA 的总 framing：为什么已有 PTQ 难以直接迁移到 VLA，尤其是带 DiT action head 的 VLA。
- **Figure 1 (p.2)**：QuantVLA 在 VLA efficiency 方法谱系中的定位图，也是 “不改架构、直接做 PTQ” 这一命题的第一锚点。
- **Method section on scale drift / ATM / OHB**：后续若补 `L2`，这里是“attention temperature drift”和“output energy drift”为何成为关键 failure mode 的最好入口。
- **Table 1 (p.7)**：Selective layer-quantization 的先导结果锚点，用来说明为什么单纯量化而不做 ATM/OHB 会失稳。
- **Table 2 (p.8)**：LIBERO 主结果锚点，用来核对 QuantVLA 在 OpenPI π0.5、GR00T N1.5 上的 success、memory 与 relative memory savings。
- **Table 3 (p.8) / Table 4 (p.9)**：分别对应 π0.5 的精度设置与 GR00T N1.5 的 denoising-step 变化，用来支撑“不同 precision / noise settings 下仍保持稳定”。
- **Table 7 (p.13)**：OpenVLA 附加结果锚点，用来判断 QuantVLA 对非-DiT action head 的外推边界。

## 待核点

- `70%` 内存节省与 “超过 full-precision baseline” 目前仍是 bundled headline，后续需要拆到具体模型、比特宽度与 benchmark 表格。
- 论文把 selective layout、ATM、OHB 一起包装成 QuantVLA 主体；后续 evidence 层可能需要区分“layout 选择本身”的贡献与“两类尺度校准”的贡献。
- 论文一方面强调自己是 DiT-based VLA 的首个成功 PTQ，另一方面又在附录中扩展到 OpenVLA；后续需要明确哪些结论是 DiT-specific，哪些只是更一般的 VLA quantization 经验。
- 若后续把它放进一般 `model compression` 路线，需要保留“主验证对象是带 DiT action head 的 VLA”这一边界。
