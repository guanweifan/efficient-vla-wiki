## Claim
- 这篇论文要解决的是：离散 diffusion VLA (`dVLA`) 虽然比离散 autoregressive VLA 具有更强的并行解码潜力，但当前推理速度仍远低于真实机器人通常需要的 `30 Hz` 实时控制频率，因此很难真正落地到物理系统。
- 作者提出 `Fast-dVLA`，核心思路是利用 dVLA 内部隐含的 `block-wise autoregressive tendency`，改写为一种可复用 `KV cache` 的 `block-wise diffusion` 解码范式：把动作序列拆成 block，按 block 逐步展开，同时在 block 内与跨 block 做并行解码，以兼顾 decoding reliability 与 throughput。
- 论文主张：通过 `KV cache reuse + inner/inter-block parallelism + asymmetric distillation` 的组合，`Fast-dVLA` 能把 dVLA 推理首次推进到接近实时区间，在保持甚至略微提升任务表现的同时显著加速，而不是像部分 acceleration 方法那样明显牺牲 success rate。
- headline numeric claims 包括：
  - 在 simulated benchmark 上，`Fast-dVLA` 相比现有 dVLA acceleration 范式取得约 `2.8×–4.1×` speedup，同时保持 action performance。
  - 在 `UD-VLA` 上，作者声称其在长时序 `CALVIN ABCD-D` 上实现约 `2.8×` inference speedup，并维持 superior performance。
  - 在 real-world 实验中，系统在三类任务上维持一致的 `30 Hz` execution frequency，并在 conveyor picking 上达到接近前人两倍的效率。
  - 在训练效率上，作者的 asymmetric distillation 方案约 `2,000` steps 收敛，约为从 finetuned weight 继续训练的 `5×` 速度、约为 from-scratch 方案所需步数的 `1/10`。

## 待核点
- `2.8×–4.1×` 是相对现有 dVLA acceleration 范式与特定 base model / benchmark 的 headline，不应直接写成所有 dVLA 都能获得的统一倍数。
- real-world 部分的 `30 Hz` 是 execution frequency headline，但 success rate / completion time 在不同任务间是 tradeoff 结构；后续不能把它写成“各任务都全面领先”。
- `Fast-dVLA` 的收益同时混合了 block-wise diffusion、KV cache reuse、pipeline decoding 与 asymmetric distillation；若后续要单独归因于某一组件，需要继续回主表与 ablation 拆分。
- `Fast-dVLA` 的主验证对象是 Dream-VLA、DD-VLA、UD-VLA 等离散 diffusion 系模型；后续若写成对一般 VLA 的通用 acceleration recipe，需要保留 model-family caveat。
