## Claim
- 页面定位：这是一篇 **quantization + pruning co-design / training-free inference compression** 论文；核心贡献是让低比特量化与 visual token pruning 在 VLA 上协同工作，而不是新的主干架构或单纯量化论文。
- 这篇论文提出 **SQAP-VLA**，试图解决一个明确的系统问题：在 VLA 上，低比特量化和 visual token pruning 并不是天然可叠加的，两者直接拼接会因为 attention / feature distribution 扭曲而导致明显性能下降。作者把自己的主要贡献定义为：第一次给出一个**结构化、training-free** 的联合框架，让量化与剪枝可以协同工作，而不是彼此破坏。来源：[[raw/2509_09090_SQAP-VLA.pdf]]，第 1 页 Abstract；第 2-3 页 Introduction。
- 核心方法主张是“co-design”而不是单点技巧：在 pruning 侧，SQAP-VLA 引入 **quantization-insensitive preservation**、**robot-aware protection**、**spatially-aware sampling**；在 quantization 侧，则通过 **Hadamard transformation** 与 **tensor-wise quantization** 提高 pruning criteria 的稳定性。来源：[[raw/2509_09090_SQAP-VLA.pdf]]，第 2-5 页 Fig. 1、Sec. 3。
- 论文保留了清晰的 headline numeric claims，但这些数字需要拆开：`1.93× speedup` 来自 W4A4 + pruning 的效率分析；`up to 4.5% average success rate enhancement` 与 `36% improvement over EfficientVLA` 又来自不同 benchmark / baseline / comparison view。更稳的写法是：**SQAP-VLA 在 CogACT-centered、training-free、W4A4 设置下证明 quantization 与 pruning 可以协同，而不是互相破坏。** 来源：[[raw/2509_09090_SQAP-VLA.pdf]]，第 1 页 Abstract；第 6-8 页 Sec. 4.2-4.4、Table 1-4、Fig. 4。
- 从当前证据看，这篇论文更像一篇**推理期压缩与系统协同设计**论文，而不是一篇新的基础 VLA 架构论文。它的主要价值在于说明量化与 pruning 的 incompatibility 以及如何在 training-free 条件下把两者重新协调起来。来源：[[raw/2509_09090_SQAP-VLA.pdf]]，第 1-3 页 Abstract、Introduction；第 6-8 页 Main Results、Efficiency Analysis。

## 待核点
- `1.93× speedup`、`4.5% average success rate enhancement`、`36% improvement over EfficientVLA` 来自不同段落和不同比较对象；后续需要把 baseline、场景和指标拆清，避免把它们误写成同一个统一 claim。
- 当前实验主体围绕 **CogACT** 和仿真任务展开，且 visual matching / variant aggregation 两类场景的表现不完全一致；后续需要决定在单篇页里如何呈现这种 benchmark dependence。
- 论文同时强调 training-free、W4A4、edge deployment、co-design；后续 taxonomy 需要决定是把它优先归入 `quantization+pruning co-design`，还是更泛的 `VLA inference acceleration`。
- `up to 4.5% average success rate enhancement` 与 Table 1 / Table 4 中的叙述存在语境差异；后续需要明确它究竟对应 visual matching、overall average，还是特定 compression-strategy ablation。
