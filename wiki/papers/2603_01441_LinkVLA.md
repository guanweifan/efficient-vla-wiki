## Claim
- 页面定位：这是一篇 **driving-specific language-action alignment + coarse-to-fine decoding** 方法论文；其核心贡献是同时处理 instruction-to-action alignment 与 autoregressive action generation latency，而不是一般 VLA 的统一推理加速框架。
- 这篇论文要解决的是：自动驾驶 VLA 虽然具备语言推理与交互潜力，但现有方法仍有两个核心问题，一是语言指令与动作轨迹之间存在 persistent misalignment，二是典型 autoregressive action generation 推理过慢，不利于实时部署。
- 作者提出 `LinkVLA`，核心思路是同时解决“对齐”和“速度”两件事：一方面把语言 token 与动作 token 放入统一离散 codebook，在结构上建立 language-action link；另一方面加入 `action understanding objective`，要求模型从轨迹反向生成描述性文本，在训练中显式强化双向语言-动作语义绑定；同时用 `coarse-to-fine (C2F)` 两阶段轨迹生成替代逐步 AR 解码。
- 论文主张：统一 token space + bidirectional action-understanding supervision + C2F decoding 的组合，能同时提升 instruction following、closed-loop driving performance 与 inference efficiency，而不是只能在三者中二选一。
- headline 数字需要拆开理解：
  - `86%` 的速度节省是相对论文内部 `AR` variant 的推理时间节省，不是相对所有 baseline 的统一加速倍数。
  - `91.01` driving score 与 `74.55%` success rate 来自 `Bench2Drive` 的 closed-loop driving 评测。
  - `87.16%` mean success 则来自 instruction-following benchmark，是另一套评测口径。
- 更稳的主张是：`LinkVLA` 通过 unified token space、action-understanding supervision 与 `C2F` decoding，把 language-action alignment 与 driving-time latency reduction 放在同一训练/推理框架里共同优化。

## 待核点
- `86%` 的速度节省是相对本论文的 `AR` variant 而不是对所有 baseline 的统一加速倍数；后续引用时要保留比较对象。

- 论文收益同时来自 unified tokenization、C2F decoding 和 action-understanding alignment；若后续把提升单独归因于某一个模块，需要继续回 Table 3 / Table 5 的 ablation。
