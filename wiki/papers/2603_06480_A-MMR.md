## Claim

这篇论文的核心命题是：VLA-based vision-language navigation 的 token pruning 不应把所有视觉 token 视为同质对象，因为 VLN 是一个显式依赖历史观测的长时序任务；如果像单帧 VLM 那样统一裁剪，就会破坏当前视角与历史记忆之间的时空关系。为此，论文提出一个 training-free 的 history-conditioned spatio-temporal visual token pruning 框架，其关键思想是**把 current frame 与 history frames 区别对待**：当前视角做 spatial token selection，历史记忆做 query-guided spatio-temporal compression，并统一由 Adaptive Maximal Marginal Relevance (A-MMR) 来平衡 saliency 与 diversity。作者 headline claim 是：在 R2R / RxR 的高 pruning ratio 条件下，该方法相较 SparseVLM、DivPrune、VisPruner 等方法保留更高的导航性能；例如在 R2R 的 `90%` pruning 下，SPL 相对 DivPrune 可提升 `17.81%`，同时延迟可比对手再低 `10.96 ms`；并已在 Unitree Go2 四足机器人上完成 real-world deployment。

显式 caveat：`17.81%`、`10.96 ms`、以及 RxR 上的其他提升数字来自不同 benchmark、不同 pruning ratio、不同比较对象，不能被读成一个统一设置下的总优势。论文真正更稳的主张是：**history-conditioned VLN 的 pruning 需要同时保留当前视角的空间覆盖和历史记忆的时空多样性**，而不是只追求单帧 token 压缩率。

## Methodology Index

- **History-conditioned VLN pruning**：明确把 VLN 看作依赖历史记忆的长时序任务，而不是单帧 reactive token pruning。
- **Current vs history split**：对当前视角做 spatial token selection，对历史视角做 spatio-temporal memory compression。
- **Adaptive Maximal Marginal Relevance (A-MMR)**：用统一迭代目标同时平衡 token saliency 与 diversity，而不是硬编码 token split。
- **Query-Guided Reweighting**：用当前帧保留下来的 query token 去重加权历史 token，构建紧凑但信息量高的 memory pool。
- **Training-free / plug-and-play integration**：不对预训练 VLA 额外 retrain，保持对既有 StreamVLN-style 模型的直接接入。
- **Performance-efficiency tradeoff framing**：论文显式把 pruning 目标写成导航性能与延迟之间的联合优化问题。

## 待核点

- `17.81%`、`10.96 ms` 等 headline 数字目前仍是 bundled summary，后续需要分别拆到 R2R、RxR、具体 pruning ratio 与具体比较对象。
- 论文把 A-MMR 与 Query-Guided Reweighting 一起包装成整体方法；后续 evidence 层可能需要区分“current-frame saliency/diversity 选择”的贡献与“history-memory reweighting”的贡献。
- 论文强调方法是 training-free、plug-and-play 的 VLN pruning 框架，但当前直接证据主要基于 StreamVLN 与 VLN benchmark；后续需要明确它向更一般 VLA navigation / non-navigation setting 的外推边界。
