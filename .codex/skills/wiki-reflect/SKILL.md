# Wiki Reflect

## 用途

这个技能用来把 `threads/` 里已经站稳的内容写回 `wiki/`。

适合这些情况：

- 某个 thread 里已经出现稳定命题
- 同一类问题反复出现，值得整理进 `wiki/`
- 现有页面缺少某个稳定比较维度或术语边界

## 不适合的情况

- 吸收新论文
- 修断链、修计数、修控制文件
- 继续扩写 thread 本身
- 做整库级重构

## 先读什么

- `AGENTS.md`
- `WIKI_SCHEMA.md`
- 相关的 `threads/*.md`
- 对应的 `wiki/` 页面

## 处理单位

默认按“可核证命题”处理，不按整篇 thread 处理。

先把 thread 里的内容拆成几类：

- 写回 `wiki`
- 继续留在 `threads`
- 还没站稳，继续待核
- 需要转交给别的技能

## 核心规则

- 写回前要回到 `raw/` 或下层页面核对。
- 优先补现有页面。
- 只有现有结构放不下，才考虑新页。
- 回写范围要小，边界要清楚。

## 建议流程

1. 先定范围：一个 thread，或者一组明显相关的 thread。
2. 拆出值得写回的命题。
3. 回原文或相关页面核对。
4. 判断最小落点：
   - 单篇页
   - 证据页
   - 主题页
5. 写回 `wiki/`。
6. 把这次 reflect 记进 `wiki/log.md`。

## 适合写回的内容

- 稳定的比较维度
- 已经核对过的跨论文共识或分歧
- 术语边界
- thread 暴露出的明显页面缺口

## 不适合写回的内容

- 一次性措辞
- 没核对过的推理
- 只在当前 thread 里成立的临时判断
- 研究建议和发散想法

## 输出

- `wiki/` 的小范围更新
- 一段简短说明，讲清：
  - 从哪些 thread 来
  - 写回了什么
  - 哪些内容还留在 `threads`
  - 哪些内容被转交

## 转交条件

- 新论文还没吸收到 `wiki/`：转 `wiki-ingest`
- 主要问题是断链、漏链、控制文件失配：转 `wiki-lint`
- 问题还在探索，暂时站不稳：留给 `wiki-query`
- 已经变成多主题重写：转周期维护

## 禁止事项

- 不直接复制 thread 文本到 `wiki/`
- 不跳过原文核对
- 不把争议点写成共识
- 不借 reflect 顺手扩写一串无关页面

## 验收

- 新写入的关键信息都能回到 `raw/` 或下层页面
- 写回范围小而集中
- 没写回的内容有去处
- `wiki/log.md` 已记录本次 reflect
