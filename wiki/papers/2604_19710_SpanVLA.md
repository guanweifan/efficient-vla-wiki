## Claim
- 页面定位：这是一篇 **autonomous-driving action-bridging / flow-matching action expert** 论文；它的核心效率问题是 autoregressive trajectory generation 随 waypoint/action length 增长带来的 latency。
- SpanVLA 的核心方法是把 VLM perception/reasoning features 通过 action bridge 接到 flow-matching action expert，并用 historical trajectory initialization 生成未来轨迹，从而避免长动作序列的逐 token autoregressive decoding。来源：[[raw/2604_19710_SpanVLA.pdf]]，Abstract、Figure 1、method section。
- 论文还引入 GRPO-based reinforcement fine-tuning 和 mReasoning 数据集，重点处理 complex reasoning scenarios、negative samples 与 recovery samples；这些设计主要服务 robustness / long-tail recovery，而不是独立定义推理效率路线。来源：[[raw/2604_19710_SpanVLA.pdf]]，Abstract、Figure 3、RFT section。
- headline 结果需要拆开理解：
  - `0.67s` action-policy total time 与 `-46% / -74%` reduction 出现在 NAVSIMv1 action policy comparison；
  - PDMS / EPDMS 是 driving benchmark performance 口径；
  - GRPO 与 negative-recovery samples 是 post-training robustness 口径。
- 更稳的主张是：SpanVLA 是 inference-time action-generation compression 的驾驶场景例子，把显式 reasoning 仍交给 VLM，但把未来 trajectory generation 转给 parallel / flow-matching action expert。
