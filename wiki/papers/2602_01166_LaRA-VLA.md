## Claim
- 页面定位：这是一篇 **latent reasoning VLA** 论文；它的核心贡献是把 textual / visual CoT 内化成连续 latent reasoning，并在推理时去掉显式 CoT generation。
- 这篇论文提出 **LaRA-VLA**，目标是在 VLA 中把显式 textual / visual CoT 内化成**连续 latent reasoning**，从而兼顾 reasoning 结构与实时控制效率。作者的核心判断是：现有 CoT-based VLA 虽然有效，但一方面推理时会生成冗长的显式中间结果，带来很高的 latency；另一方面，无论文字 CoT 还是离散视觉 token CoT，都与机器人连续感知和连续控制存在表征错配。来源：[[raw/2602_01166_LaRA-VLA.pdf]]，第 1-2 页 Abstract、Introduction、Fig. 1。
- 方法上的主张是一个 **unified latent-reasoning VLA**：同时把 textual CoT 和 visual reasoning internalize 到 **continuous latent representations** 中，在 latent space 里完成 reasoning 与 prediction，并在推理阶段完全消除显式 CoT generation。为实现这一点，论文引入了一个 **curriculum-based training paradigm**，逐步从显式 textual/visual CoT 监督过渡到 latent reasoning，再把 latent reasoning dynamics 适配到 action generation。来源：[[raw/2602_01166_LaRA-VLA.pdf]]，第 1 页 Abstract；第 2-5 页 Introduction、Fig. 2、方法章节。
- 论文还强调训练数据与训练稳定性设计：作者构建了 **LIBERO-LaRA** 与 **Bridge-LaRA** 两个结构化 CoT 数据集，并用 **EMA visual encoder** 作为 target network 稳定 latent 表示学习，避免表示坍塌。当前这些内容更像方法能否成立的关键配套，而不是附属工程细节。来源：[[raw/2602_01166_LaRA-VLA.pdf]]，第 2 页贡献段；第 4-6 页方法、数据与训练描述。
- 论文保留了清晰的 headline efficiency/performance claims，但需要拆开理解：
  - `up to 90%` 是相对显式 CoT 方法的 latency reduction 口径；
  - `135 ms latency` 是更具体的单项推理延迟结果；
  - simulation benchmark 与 long-horizon real-robot 的优势又属于任务表现口径，不应与前述 latency 数字混成单一 superiority claim。来源：[[raw/2602_01166_LaRA-VLA.pdf]]，第 1 页 Abstract；第 8-10 页实验结果与总结。
- 更稳的主张是：`LaRA-VLA` 试图把显式 CoT 的结构收益保留下来，同时把 reasoning 迁移到连续 latent space，从而兼顾 action-oriented reasoning 与实时控制效率；但其训练仍依赖显式 CoT / visual prediction supervision。

## 待核点
- `up to 90% latency reduction`、`135 ms`、以及 simulation / real-robot 上的性能提升来自不同评测层面；后续需要拆清具体 baseline 和指标口径。
- 论文标题是 `Latent Reasoning VLA`，但仓库里文件名采用 `LaRA-VLA`；后续需要统一命名口径，避免 `LaRA-VLA` 与 `Latent Reasoning VLA` 并存引发混淆。
- LaRA-VLA 同时强调 `latent reasoning paradigm`、`curriculum-based training`、`continuous latent CoT` 和 `action adaptation`；后续 taxonomy 需要统一其主定位。
- 当前方法仍依赖显式 CoT 与视觉预测作为训练阶段 supervision，只是在 inference 阶段去掉显式 CoT；后续需要决定在单篇主 claim 中是否更显式写出这一点，避免误读为完全 teacher-free latent reasoning。
- 若后续把它放进一般 `CoT efficiency` 路线，需要保留“核心是 latent internalization，而不是单纯去掉 CoT token”这一边界。
