## Claim
- 页面定位：这是一篇 **hierarchical slow-VLM + fast-policy control system** 论文；核心贡献是用异步 latent handoff 提高控制频率，而不是直接让 VLM 端到端输出高速动作。
- 这篇论文聚焦 VLA / VLM 后端带来的低控制频率和高延迟问题，目标是在尽量保留泛化能力的同时，把系统推到更适合动态操作的控制速度。
- 作者提出 `HiRT`，把慢速、高层语义理解的 VLM 与高速、低层动作执行策略做成层级式异步系统：VLM 低频更新 latent，轻量视觉策略高频读取 latent 并连续输出动作。
- 论文主张这种层级拆分能把“大模型的语义理解”和“小模型的实时控制”结合起来，而不是直接让 VLM 端到端输出低层动作。
- headline 结果需要拆开理解：
  - `4.1 Hz -> 9.8 Hz` 来自 **Table 1 的 quasi-static task** 结果，配套结论是“速度显著提高且平均表现大体可比”，不是动态任务 headline。
  - `48% -> 75%` 来自 **Table 2 的 real-world dynamic task** 平均成功率，对应的是另一组 evaluation layer。
  - “在动态真实任务上完成时间最短”也是 Table 2 的系统层结果，不应和 Table 1 的频率数字压成同一 operating point。
- 更稳的写法是：HiRT 的 headline 必须至少拆成 **quasi-static 频率/性能折中** 与 **dynamic real-world 成功率/完成时间** 两层；尤其静态真实任务并不是全线优于 Vanilla-VLA，不能压成统一 superiority claim。

## 待核点
- `double the control frequency` 在正文主表里对应的是 `4.1 Hz -> 9.8 Hz`，更接近 `2.39x`；后续如果要引用“翻倍”这一说法，需区分 abstract headline 与具体实验设置。
- 静态真实任务里，HiRT 的平均值 `70.0` 低于 Vanilla-VLA 的 `71.3`；后续若写“整体优于 baseline”，必须明确区分 simulation、static real-world、dynamic real-world 三类结果，也不要把动态场景的系统收益外推到全部 setting。
- Table 1 的 `Speed/Hz` 与 Table 2 的 `Time/s` 分别对应控制频率和任务完成时间；后续不能把二者都写成同一种 latency/speedup 指标。
- 文中同时讨论多种 conditioning 方式和多种视觉输入，后续若要归纳“HiRT 的关键增益来自哪里”，需要进一步回到 Table 3 和 appendix 做更细证据拆分。
- 异步频率与 latent stale 问题在主文只给出机制描述；若后续要把“异步采样”单独做成方法点，需要进一步补 appendix 中关于 asynchronous sampling 的实验细节。
