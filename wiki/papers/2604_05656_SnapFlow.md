## Claim
- 页面定位：这是一篇 **single-step flow-matching action generation via self-distillation** 论文；它的主贡献是把 flow-matching VLA 的 iterative denoising 压成 `1-NFE` 单步生成，而不是继续优化每一步的 token 数量或 backbone 层数。
- 这篇论文要解决的是：在 `π0 / π0.5 / SmolVLA` 这类 flow-matching VLA 中，10-step Euler denoising 本身就是主要 latency 瓶颈，直接把步数压低又会显著损伤成功率。作者把问题写成 velocity field 没有为 single-step jump 校准。来源：[[raw/2604_05656_SnapFlow.pdf]]，Abstract；Introduction。
- `SnapFlow` 的核心方法是 progressive self-distillation：把标准 flow-matching 样本与 two-step Euler shortcut consistency 样本混合训练，并用 target-time embedding 在同一架构里同时支持 local velocity estimation 与 global one-step generation。来源：[[raw/2604_05656_SnapFlow.pdf]]，Abstract；Sec. 3.4。
- headline 数字需要拆开理解：在 `π0.5` 上，论文报告 `98.75%` average success 对比 10-step teacher 的 `97.75%`，同时给出 `9.6×` denoising speedup 与 `274 ms -> 83 ms` end-to-end latency reduction；在 `SmolVLA` 上又报告 `3.56×` end-to-end acceleration 与 `8.3%` MSE reduction。不同模型家族与指标不能并成一条统一结论。来源：[[raw/2604_05656_SnapFlow.pdf]]，Abstract；Introduction。
- 更稳的主张是：`SnapFlow` 把 flow-matching VLA 的 action-head sampling 从多步迭代压到单步生成，重点是 action decoding compression，而且它明确强调自己与 layer distillation、token pruning 路线正交。

## 待核点
- `98.75%`、`97.75%`、`9.6×`、`274 ms -> 83 ms`、`3.56×`、`8.3%` 分别对应不同模型和指标层，后续需要逐一拆分。
- 论文强调无需 external teacher 和 architecture change；后续若要和一般 consistency distillation 方法比较，需要继续核对这种 minimal intervention 的边界。
- 当前 strongest evidence 主要在 `π0.5` 与 `SmolVLA`；后续不能写成对所有 flow-matching VLA 都已统一验证。
