## Claim
- 页面定位：这是一篇 **flow-based continuous-action generation / self-correction** 论文；核心贡献是把 action generation 从同步 flow matching 改写成带 confidence-guided regeneration 的两阶段过程，而不是一般性的 VLA 效率 benchmark 论文。
- 这篇论文要解决的是：基于 flow matching 的 VLA 往往采用 rigid、uniform 的同步生成时间表（synchronous FM），缺乏对动作上下文和生成置信度的利用，因而在长时程或高精度任务中容易因为一次错误动作而级联失败。
- 核心主张是：把 action generation 改写成“先同步初生成、再异步自纠正”的两阶段过程，并让第二阶段只重生成低置信度 action token，就能在 continuous action generation 场景下引入 self-correction；再通过 unified training 让同一个模型同时支持 `SFM` 与 `AFM`，并在 AFM 推理阶段复用 VL KV-cache。
- headline 数字需要拆开理解：
  - `97.4%` 是 `LIBERO` 四个 task suite 合并微调后的平均 success rate；
  - `70.8%` 是 `WidowX / SimplerEnv` 平均 success rate；
  - `74.9% / 63.3%` 分别是 `Google Robot / SimplerEnv` 在 visual matching / variant aggregation 两种 protocol 下的平均 success rate；
  - 这些不是一个统一“总体平均”。
- 更稳的主张是：**AsyncVLA 在 flow-based VLA 内部，把 self-correction、confidence-aware regeneration 和 unified training 结合起来，在多个 embodied benchmark 上取得更强的整体平均表现；其效率收益主要体现为同一模型内部 AFM 相对 vanilla SFM 的 cache reuse 与 data efficiency，而不是跨模型统一 latency headline。**

## 待核点
- 这篇没有一个统一的单一 headline 数字，而是跨 `LIBERO`、`WidowX`、`Google Robot` 三类评测；后续不能把不同 benchmark 的 success rate 直接压成一个泛化能力结论。
- `AsyncVLA` 的效率主张更多体现在 “better data efficiency / unified training / KV-cache utilization / AFM 比 SFM 更快”，而不是像纯加速论文那样给出统一 latency headline；后续若写“efficient”需要明确是训练利用效率与生成机制效率，不是单一 wall-clock headline。
- `Google Robot` 上 `π0` 在某些单项任务上略高于 AsyncVLA，`WidowX` 上也不是所有子任务都绝对领先；因此整体应写成“整体平均最好 / generally strongest”，而不是“每项都 SOTA”。
- `w/o Unified Training` 在消融中只有 `7.3%` 平均成功率，说明 unified training 对 AFM/SFM 对齐至关重要；后续如果把自纠正能力归因给 AFM alone，需要保留 unified training 是必要前提。
