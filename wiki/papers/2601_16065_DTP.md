## Claim
- 页面定位：这是一篇 **task-irrelevant attention correction / distracting token pruning** 论文；它的核心贡献是改善视觉注意力模式并探索性能上界，而不是单纯为了提速做 token pruning。
- 这篇论文要解决的是：VLA 在 manipulation 中可能把大量注意力放到 task-irrelevant 图像区域上，即作者所说的 `distracting tokens`。这些 token 会在 action generation 时干扰模型，把动作朝错误区域引导，降低任务成功率。
- 作者提出 `Distracting Token Pruning (DTP)`，这是一个 training-free、plug-and-play 的推理期框架。它先通过 prompt-to-visual relevance 构造重要区域，再分析 action generation 时的视觉注意力热图，最后用一个 intersection-based pruning 规则，只在“不重要区域里却获得过高注意力”的 token 上做 pruning。
- 论文主张：很多 VLA 失败并非来自模型能力上限，而是来自 attention leakage 到 task-irrelevant region；如果在不改架构、不加额外输入的前提下动态裁掉这些 distracting tokens，任务成功率可以稳定提升，并可用来探索模型在当前架构下的性能上界。
- headline 数字需要拆开理解：
  - 在 `SIMPLER WidowX Robot` 上，`SpatialVLA 29.2% -> 37.5%`、`Nora 6.2% -> 11.5%`、`UniVLA 68.7% -> 74.0%` 是一组特定模型/平台结果；
  - `SIMPLER Google Robot` 上是另一组较小提升；
  - `LIBERO` 上则主要是 `Nora + DTP` 在 `LIBERO-10` 上约 `+6.6%`、其余 suite `+1.4%` 到 `+2.6%`。  
- 更稳的主张是：`DTP` 通过裁掉 task-irrelevant 但被过度关注的 distracting tokens，改善 VLA 的视觉注意力模式，并据此探索当前架构下的性能上界；它不应被简单归入一般的效率 pruning recipe。

## 待核点
- 当前 headline 里的提升值主要来自特定模型与特定 benchmark，不能写成统一“所有 VLA 都能获得某个固定收益”。
- `LIBERO` 结果只在 `Nora` 上额外验证；后续若写“DTP 在 LIBERO 上普遍有效”，需要保留模型范围 caveat。
- DTP 的目标更接近“改善 visual attention / 探索当前架构 performance upper bound”，不等于通用 token pruning recipe；后续主题归类要注意与一般 efficiency pruning 区分。
- tolerance `τ` 对效果影响很大，且不同模型的最优 `τ` 不同；后续若写“DTP 可稳定提升”，应同时保留超参数敏感性这一 caveat。
- 若后续把它放进一般 `token pruning` 路线，需要继续保留“目标是 attention correction，不是主打 latency gain”这一边界。
