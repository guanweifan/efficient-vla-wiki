## Claim
- 页面定位：这是一篇 **training-free adaptive model pruning + orchestration** 论文；它的核心贡献是根据环境动态在线更新 backbone sparsity，并把 pruning overhead 隐藏进 inference bubbles。
- EcoVLA 的核心主张是：VLA 的最优模型稀疏模式会随着物理环境演化而动态变化，因此 static pruning 不够自适应，而固定间隔 dynamic pruning 又过于粗糙且往往依赖额外训练。
- 论文据此提出一个 training-free、plug-and-play 的 adaptive model pruning 框架，用于在线更新 VLA backbone 的稀疏模式，并且尽量把 pruning overhead 隐藏在原本的 inference bubbles 里。具体来说，EcoVLA 由两部分组成：Environment-aware Adaptive Pruning (`EAP`)，负责根据当前环境和时间一致性更新 channel sparsity pattern；Interleaved Inference Orchestration (`I2O`)，负责把 pruning 计算与推理过程并行交织，避免直接把 pruning 开销叠加到 wall-clock latency 上。
- headline 数字需要拆开理解：
  - `1.60x` / `0.4%` 对应单独使用 EcoVLA 时的效率-精度折中；
  - `2.18x` / `0.5%` 对应与 token pruning 组合后的结果；
  - OpenVLA-OFT、`π0.5`、`CogACT` 与 real robot 则属于不同模型和 deployment setting，不应压成统一倍率。  
- 更稳的主张是：**environment-aware adaptive model pruning + orchestration** 可以在 training-free 前提下取得更强的效率-精度折中，但其优势建立在环境动态变化与 pruning overhead 可被交织隐藏这两个条件之上。

## Methodology Index

- **Environment-aware Adaptive Pruning (EAP)**：根据当前环境变化与视觉动态在线更新 pruning pattern，而不是用静态 calibration sparsity。
- **Temporal Consistency Pruning**：通过融合 instantaneous feature 与 historical feature，保持 VLA 执行过程中的时序一致性，减少 sparsity lag 或抖动。
- **Adaptive channel pruning**：聚焦 VLM backbone 的 structured channel pruning，而非只做 token-level 优化。
- **Interleaved Inference Orchestration (I2O)**：利用 VLA inference 中的 FLOPs bubbles，将 pruning computation 并行塞入，不阻塞主推理链路。
- **Training-free / plug-and-play composition**：既可单独工作，也可与 token pruning 等其他 VLA acceleration 方法正交组合。
- **Cross-model evaluation framing**：论文明确在 OpenVLA-OFT、π0.5、CogACT 以及 real robot 上验证。

## 待核点

- `1.60x`、`2.18x`、`0.4%`、`0.5%` 这些 headline 数字来自不同模型、不同 pruning ratio 以及是否组合 token pruning 的不同设置，后续需要逐项拆证据。
- 论文把 EAP 与 I2O 一起包装成 EcoVLA 的总创新；后续 evidence 层可能需要区分“adaptive pruning quality”的收益与“overhead masking / orchestration”的收益。
- 论文强调可与 token pruning 正交组合，但这种组合优势当前主要通过少数特定方法与设置呈现；后续需要明确其组合边界，而不是泛化成对任意 acceleration stack 都同样有效。
- 若后续把它放进一般 `model pruning` 路线，需要保留“核心贡献是 environment-aware adaptation + orchestration，而不是静态 sparsity 设计”这一边界。
