## Claim
- 页面定位：这是一篇 **kinematics-rectified speculative decoding for VLA inference** 论文；其核心贡献是把运动学预测与动态 acceptance threshold 引入 speculative decoding，而不是提出新的 VLA policy architecture。
- 这篇论文要解决的是：把 `speculative decoding (SD)` 用到 VLA 上时，现有做法仍有两个主要瓶颈，一是 token error 需要依赖高成本 `re-inference` 修正，二是 relaxed acceptance threshold 很难稳定设定，导致 speed-accuracy trade-off 不稳。
- 作者提出 `KERV`，核心是把 token-domain VLA 与 kinematic-domain prediction 结合起来：用基于 `Kalman Filter` 的运动学预测补偿 SD 误差，从而减少昂贵的 re-inference；再用基于 `kinematic variability` 的动态阈值调整替代固定 acceptance threshold。
- 论文主张：如果把机器人运动学显式引入 VLA speculative decoding，就能在尽量不损失 success rate 的前提下，获得比 naive VLA+SD 更高的推理加速，并相对 `SpecVLA` 进一步改善 speed-accuracy Pareto。
- headline 数字需要拆开理解：
  - 相比 `SpecVLA`，`KERV` 在 diverse tasks / environments 上获得约 `27%∼37%` acceleration，但这组数字对应的是特定对手与特定实验配置。
  - 相比 naive `VLA+SD`，`KERV` 达到约 `1.48×∼1.57×` speedup，这是一组不同比较口径下的 headline。
  - “nearly no SR loss” 也不是绝对零损失，而是在部分环境里仍存在小幅 `SR` tradeoff。
- 更稳的主张是：`KERV` 证明 VLA speculative decoding 若显式利用机器人运动学做误差补偿和阈值调节，就能比纯 token-domain SD 更稳定地改善 speed-accuracy tradeoff。

## 待核点
- `27%∼37% acceleration` 是相对 `SpecVLA` 的 headline；`1.48×∼1.57× speedup` 则是相对 naive `VLA+SD`，后续不能把两组数字混成同一比较口径。
- “nearly no SR loss” 是整体摘要式说法，但正文也提到相对 `SpecVLA` 在部分环境上仍有约 `1.5%` 和 `0.4%` 的 `SR` 下降；后续引用时要保留这种 tradeoff。
- 当前主实验建立在 `LIBERO` 四个 task suites 与作者的实现配置上；推广到更广泛 real-world robotic arms，作者自己也保留了泛化 caveat。
- `KERV` 的收益同时混合了 KF-based compensation、kinematic-based threshold adjustment 与 `CPU+GPU` implementation；若后续把提升单独归因于某一组件，需要继续回 Table 3 / Table 4 拆分。
