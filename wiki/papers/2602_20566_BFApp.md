## Claim

这篇论文的核心命题是：多视角 VLA 的效率问题不能只靠通用 VLM token pruning 思路解决，因为机器人 manipulation 中真正重要的视觉信息同时受到 **视角间动态重要性** 和 **视角内任务相关区域** 两层结构影响；如果 pruning 忽略这两层差异，就会在加速的同时丢掉操作关键线索。为此，论文提出 BFA++，一个专门面向 multi-view VLA 的 hierarchical dynamic token pruning 框架。它先用 Intra-IP 在每个视角内部筛选 task-relevant token，再用 Inter-IP 对不同视角的重要性做阶段性评估，最后进行 local-to-global hierarchical pruning。作者 headline claim 是：在 RoboTwin 与 real-world manipulation 上，BFA++ 相比基线方法可同时提高速度与成功率；在 `π0` 上约有 `1.8x` speedup 和 `~10%` success gain，在 `RDT` 上约有 `1.5x` speedup 和 `~10%` success gain。

显式 caveat：`1.8x`、`1.5x` 与 `~10%` 提升来自不同 backbone、不同环境与不同比较对象，不应被读成一个统一设置下的固定收益。论文更稳的命题是：**multi-view VLA 的 pruning 需要 task-aware、hierarchical、view-sensitive 的 token 选择机制**，而不是简单把通用 token pruning 套到机器人场景上。

## Methodology Index

- **Hierarchical token pruning**：先做局部 view 内 pruning，再做跨 view 全局 pruning，而不是单阶段裁剪。
- **Intra-view Importance Predictor (Intra-IP)**：识别单个视角内部与操作相关的区域，如 gripper、target object 等。
- **Inter-view Importance Predictor (Inter-IP)**：估计不同 manipulation phase 中各 camera view 的重要性变化。
- **Local-global pruning pipeline**：先局部去噪，再按跨视角联合重要性排序，删除全局最不关键 token。
- **Supervised post-training for VLA**：通过专门的 annotation system 提供 inter-view / intra-view supervision，并在 VLA post-training 中联合训练 importance predictors。
- **Plug-and-play multi-backbone framing**：论文强调方法可用于 `π0` 与 `RDT`，不是绑定单一 backbone。

## 待核点

- `1.8x`、`1.5x` 与 `~10%` 提升目前仍是 bundled headline，后续需要分别拆到 `π0`、`RDT`、RoboTwin、real-world 和 OOD setting。
- 论文把 Intra-IP、Inter-IP 与 hierarchical pruning 一起包装成 BFA++ 主体；后续 evidence 层可能需要区分“importance supervision 有效”与“hierarchical pruning 结构有效”。
- 论文强调方法是 plug-and-play multi-backbone token pruning，但当前直接证据主要集中在 `π0` 与 `RDT` 的 multi-view setting；后续需要明确它对其他 VLA 架构或非多视角设定的适用边界。
