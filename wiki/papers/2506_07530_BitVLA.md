## Claim
- 页面定位：这是一篇 **native 1-bit VLA / low-bit end-to-end compression** 论文；核心贡献是把低比特约束放进训练与蒸馏流程，而不是后处理量化插件。
- 这篇论文要解决的是：现有 VLA 即使性能强，也往往体积大、显存占用高、端到端延迟高，不适合部署到受限的边缘机器人平台。
- 作者提出 `BitVLA`，宣称这是首个 fully native `1-bit` VLA：语言骨干和最终模型参数采用 ternary 表示 `{−1, 0, 1}`，同时把视觉编码器也压到 `1.58-bit weights + INT8 activations`。
- 论文的核心主张不是“后处理量化一下”，而是把低比特约束放进训练流程里，通过 `Quantize-then-Distill` 在多模态训练之后继续压缩视觉编码器，并用 teacher 做表征对齐。
- headline numeric claims 包括：
  - 模型推理显存约 `1.4 GB`。
  - 相比 `OpenVLA-OFT`，模型 memory `11.0×` 更小、端到端 latency `4.4×` 更低。
  - 在 LIBERO 上，BitVLA 与更大的 `OpenVLA-OFT` 总体接近；在若干 real-world 任务上，文中给出与 `OpenVLA-OFT` 可比、且优于 `π0` 的表现。
  - 论文还强调 `341.1 Hz throughput / 73 ms latency` 的推理效率结果。
  - 这些数字混合了 memory、latency、throughput、simulation 与 real-world 结果，后续需要始终拆开写。

## 待核点
- 文中同时使用 “与 OpenVLA-OFT 可比” 和 “优于某些 baseline” 两类表述；后续若写稳定结论，需要按 simulation、real-world base tasks、OOD tasks 分开，不宜压成单句。
- `1.4 GB`、`73 ms`、`341.1 Hz` 等 efficiency 指标是在指定硬件和固定 query 设置下测得；后续若做横向效率比较，需要保留硬件/benchmark 条件。
- `BitVLA` 与 `NORA-Long` 类似，论文里也有“不同 setting 下不同优势”的结构；后续如果拿它和全精度大模型比较，必须区分 “总体接近” 与 “某些具体任务超过/落后”。
- 论文多处把 `OpenVLA-OFT` 作为主要 full-precision 参照物；如果后续 chief editor 要把 BitVLA 放到更宽的“efficient VLA”主题里，可能需要决定是否将 `post-training quantization baseline` 单独视为一类比较对象。
