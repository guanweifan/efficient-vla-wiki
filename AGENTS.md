# AGENTS.md

## 1. 作用范围与优先级
1. 本文件定义本仓库的人机协作契约，作用范围为仓库根目录及其全部子目录。
2. 规则优先级：**用户当前明确指令 > 本文件 > `WIKI_SCHEMA.md`**。
3. `WIKI_SCHEMA.md` 的编辑权在 agent；只要 agent 认为 wiki 的结构、字段或流程值得更新，就应主动修改，无需征询用户；用户默认不介入该文件。

## 2. 三层目录契约
1. `raw/`：原始论文层。只存放原始 PDF 等源材料；默认只读；是唯一事实来源。
2. `wiki/`：持久知识层。由 agent 负责创建、维护、重构与交叉引用。
3. `threads/`：问题讨论层。用于记录具体问题、回答、分歧、未决项与阶段性结论。

## 3. 取材边界
1. `wiki` 与 `threads` 中的事实性内容只能取材于 `raw/`。
2. 不得把外部项目、网页、代码仓库、旧 taxonomy 或常识性印象当作 wiki 的事实来源。
3. 解析器输出、文件名、历史讨论和旧页面只可作为定位线索或工作中间层，不可替代原始证据。

## 4. 协作准则
1. 回答问题时，优先读取 `wiki/`；只要结论关键、细节敏感或 `wiki` 不足，就回到 `raw/` 复核。
2. `threads/` 可以承载推理、中间判断与比较，但它本身不是事实来源。
3. 只有已回到 `raw/` 核证的稳定结论，才可进入 `wiki/`。
4. 不把猜测写成结论；证据不足时，明确说明“不确定”或“待核”。
5. 不凭空补全作者未说过的观点、实验设置、因果解释或横向结论。
6. 发现论文之间冲突时，先记录冲突，不强行统一口径。

## 5. 变更权限
1. `AGENTS.md` 由用户主导；agent 若认为本文件需要修改，必须先征得用户同意。
2. `WIKI_SCHEMA.md` 不受前条约束，由 agent 持续主动维护。
3. agent 可在不违背本文件的前提下，自主调整 `wiki/` 结构并更新 `WIKI_SCHEMA.md`。
4. 若用户当前指令与既有结构冲突，以用户当前指令为准。

## 6. 何时应暂停
1. `raw/` 中证据不足以支持结论时，应暂停扩写，先说明缺口。
2. `wiki/` 结构若妨碍回答质量，可以先调整 `wiki`，再回答 `threads`。
3. 若局部规则与当前任务冲突，应先以本文件与用户当前指令为准。
