## Claim
- 页面定位：这是一篇面向 **离散 diffusion VLA** 的 decoding / system acceleration 论文；其主贡献是把 dVLA 的 action decoding 推到接近实时控制区间，而不是提出对一般 VLA 普遍适用的统一加速 recipe。
- 这篇论文要解决的是：离散 diffusion VLA (`dVLA`) 虽然比离散 autoregressive VLA 具有更强的并行解码潜力，但当前推理速度仍远低于真实机器人通常需要的 `30 Hz` 实时控制频率，因此很难真正落地到物理系统。
- 作者提出 `Fast-dVLA`，核心思路是利用 dVLA 内部隐含的 `block-wise autoregressive tendency`，改写为一种可复用 `KV cache` 的 `block-wise diffusion` 解码范式：把动作序列拆成 block，按 block 逐步展开，同时在 block 内与跨 block 做并行解码，以兼顾 decoding reliability 与 throughput。
- 论文主张：通过 `KV cache reuse + inner/inter-block parallelism + asymmetric distillation` 的组合，`Fast-dVLA` 试图同时改善 dVLA 的 decoding throughput、部署频率与训练适配成本；这些收益是并列结果，不应压成一个统一的“全面更快更强”结论。
- headline numeric claims 需要拆开理解：
  - **simulation speed-performance claim**：在特定 dVLA acceleration 对比、特定 backbone / benchmark 上，作者报告约 `2.8×–4.1×` speedup，同时尽量维持 action performance。
  - **specific benchmark claim**：在 `UD-VLA` 的 `CALVIN ABCD-D` 设置上，作者报告约 `2.8×` inference speedup，并声称性能仍优。
  - **real-world deployment claim**：在 real-world 实验中，系统报告三类任务都能维持 `30 Hz` execution frequency；其中 conveyor picking 的效率提升是特定任务 headline，不应外推到所有任务。
  - **training-efficiency claim**：`asymmetric distillation` 约在 `2,000` steps 收敛，这属于训练适配效率，不应和推理 speedup 写成同一层结论。

## 待核点
- `2.8×–4.1×` 是相对现有 dVLA acceleration 范式与特定 base model / benchmark 的 headline，不应直接写成所有 dVLA 都能获得的统一倍数。
- real-world 部分的 `30 Hz` 是 execution frequency headline，但 success rate / completion time 在不同任务间是 tradeoff 结构；后续不能把它写成“各任务都全面领先”。
- `Fast-dVLA` 的收益同时混合了 block-wise diffusion、KV cache reuse、pipeline decoding 与 asymmetric distillation；若后续要单独归因于某一组件，需要继续回主表与 ablation 拆分。
- `Fast-dVLA` 的主验证对象是 Dream-VLA、DD-VLA、UD-VLA 等离散 diffusion 系模型；后续若写成对一般 VLA 的通用 acceleration recipe，需要保留 model-family caveat。
- 若后续要使用“首次接近实时”这类表述，需要先拆清它对应 decoding throughput、execution frequency 还是特定 real-world task efficiency。
