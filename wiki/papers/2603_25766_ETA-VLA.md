## Claim
- 页面定位：这是一篇面向 autonomous driving 的 **temporal compression + intra-LLM token sparsification** 论文；它的核心贡献是同时压缩多帧多视角历史与 LLM 内部视觉 token，而不是单纯复用已有 cache 或只做静态 token pruning。
- 这篇论文要解决的是：driving VLA 为了利用多帧、多视角历史做时序判断，会把视觉 token 数量推得很高，进而把 LLM self-attention 的计算开销迅速放大。作者把这个问题写成 driving VLA 的时空冗余，而不是一般 VLM 的单帧 token 冗余。来源：[[raw/2603_25766_ETA-VLA.pdf]]，Abstract；Introduction。
- `ETA-VLA` 的主方法由两个互补模块组成：`Temporal Fusion Module (TFM)` 先在进入 LLM 之前压缩历史视觉特征，`Intra-LLM Sparse Aggregator (ILSA)` 再在 LLM 内部用 text-guided、RoPE-free 的语义打分与 diversity-preserving recycling 做细粒度稀疏化。来源：[[raw/2603_25766_ETA-VLA.pdf]]，Introduction；Sec. III-B/C。
- headline 数字需要拆开理解：论文在 `NAVSIM v2` 上报告了大约 `32%` 计算 FLOPs 降低，同时又给出 `85%` visual token pruning、`61%` inference FLOPs reduction、以及保留 `94%` 原始精度这一组更细的效率口径。这些数字不在同一层，不能压成统一 superiority claim。来源：[[raw/2603_25766_ETA-VLA.pdf]]，Abstract；Sec. IV-B。
- 更稳的主张是：`ETA-VLA` 通过“先做时间维压缩，再做 LLM 内语义稀疏化”的双层计算分配，把 driving VLA 的多帧多视角 token bloat 收紧到可部署范围，同时尽量保留关键驾驶语义。

## 待核点
- `32% FLOPs reduction`、`85% visual token pruning`、`61% inference FLOPs reduction`、`94% original accuracy` 分别对应哪些设置与比较基线，后续需要拆清。
- `ETA-VLA` 同时覆盖 temporal fusion 和 token sparsification；后续若要把它归到单一路线，需要保留“双层压缩”的边界。
- 当前最强证据集中在 `NAVSIM v2` driving setting；后续不能把它直接外推成对 manipulation VLA 同样成立的通用 recipe。
