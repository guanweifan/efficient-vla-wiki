## Claim
- 页面定位：这是一篇 **training-free temporal KV reuse / token caching for VLA inference** 论文；核心贡献是跨帧复用静态视觉 token 的缓存，而不是单帧 token pruning 或重新训练压缩模型。
- 这篇论文提出 **VLA-Cache**，把它定位成一种 **training-free / plug-and-play** 的 VLA 推理加速方法：核心做法不是改模型结构或重新训练，而是在相邻帧之间复用静态视觉 token 的缓存 key-value，从而减少重复计算。
- 作者的核心命题不是“所有静态 token 都能安全复用”，而是：**只有在“视觉上静态”且“任务上不关键”时，token 复用才成立**。因此方法分成两步：先找静态 token，再通过 decoder attention 过滤掉 task-relevant token，并进一步按层自适应调节复用比例。
- 论文把自己的主要收益表述为：在 `LIBERO`、`SIMPLER` 和真实机器人设置中，**最高可达约 1.7x CUDA latency speedup 与约 15% control frequency 增益，同时任务成功率基本保持**。更稳的写法是：这些 headline 来自不同模型和 setting，不应压成统一 operating point。  
  来源：[[raw/2502_02175_VLA-Cache.pdf]]，第 1 页，Abstract。
- 在更具体的模拟结果里，作者声称相对标准 `OpenVLA`，`VLA-Cache` 在 `LIBERO` 上可实现 **27.31% FLOPs reduction** 与 **1.63x latency improvement**，同时平均 success rate 仅下降 **0.3%**；并且对 `OpenVLA-OFT` 这类更高频 VLA 仍然有增益。  
  来源：[[raw/2502_02175_VLA-Cache.pdf]]，第 8 页，Sec. 5.3 与 Table 2。

## Methodology Index
- temporal token caching / temporal KV reuse
- static token selection via inter-frame patch similarity
- task-relevance filtering via decoder attention
- layer-adaptive token reuse based on attention concentration / entropy
- training-free inference acceleration
- cross-frame reuse instead of single-frame pruning/merging
- compatibility claim with `OpenVLA` / `OpenVLA-OFT` / `CogAct`

## 待核点
- `1.7x` 与 `15%` 是 abstract 里的 headline 数字，但正文中不同模型/场景下对应的具体度量并不完全相同；后续 `L2` 需要拆成按模型与 setting 的证据。
- 当前页面把 `OpenVLA`、`OpenVLA-OFT`、`CogAct` 上的效果合并在同一层概述里；后续如果要做更细的 `claims/tables` 页面，需要分模型整理。
- 真实机器人部分使用的是 fine-tuned `OpenVLA` policy 作为被加速对象；后续若要强调“training-free”的边界，需要明确区分“加速方法不训练”与“底座 policy 本身可能已被 fine-tune”，也不要把 temporal caching 和单帧 pruning 混成一类方法。
