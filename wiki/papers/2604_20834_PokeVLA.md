## Claim
- 页面定位：这是一篇 **compact backbone / lightweight VLA design** 论文；它的核心效率来源是 pocket-sized VLA foundation model，而不是动态 routing、token pruning 或推理调度。
- PokeVLA 的核心路径是两阶段：先用约 `2.4M` multimodal embodied samples 预训练 compact PokeVLM，再通过 multi-view goal-aware semantics learning、geometry alignment 和 action expert 把 manipulation-relevant representation 注入 action learning。来源：[[raw/2604_20834_PokeVLA.pdf]]，Abstract、Introduction、Figure 3。
- 论文强调的表示学习组件包括 goal-aware segmentation / semantic target learning、多视角几何对齐、以及 action queries；这些组件主要服务小模型在 manipulation 中获得足够的 spatial / semantic grounding。来源：[[raw/2604_20834_PokeVLA.pdf]]，method section、Figure 3。
- headline 结果需要拆开理解：
  - `1.22B` 是 VLM backbone/model scale 口径；
  - LIBERO / LIBERO-Plus 是 simulation benchmark performance 口径；
  - real-world xArm7 实验是部署验证口径；
  - “will open-source” 是作者承诺，需后续核验仓库状态。
- 更稳的主张是：PokeVLA 是 model-and-representation efficiency 中 compact VLA design 的新增例子，说明“小底座 + embodied/world knowledge guidance + multi-view spatial alignment” 可以作为轻量化 VLA 的一条路线。

## 待核点
- PokeVLA 的效率定位目前主要来自 compact model scale；页面未把它写成已验证 edge/on-device deployment。
- 作者声明将开源 code、model weights 和 scripts；后续如需判断 reproducibility，需要单独核验实际仓库与 release 状态。
- 该论文同时强调 world knowledge guidance、spatial grounding、multi-view geometry 与 action expert；taxonomy 中应按首要效率瓶颈归入 compact VLA design，而不是把每个组件都升格成独立主类。
