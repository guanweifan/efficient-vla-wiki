## Claim
- 页面定位：这是一篇 **training-free, observation-only visual token compression** 论文；它的核心贡献是用 `Expand + Merge` 重建前景再压缩，而不是依赖 temporal buffer 或 retrained query。
- TEAM-VLA 将自己定位为一种 **training-free、observation-only** 的 VLA visual token compression 方法，目标是在不依赖 retraining、可学习查询或历史帧缓存的前提下，改善 VLA 的 success-latency tradeoff。论文的核心主张是：已有 VLA / VLM token pruning 方法要么依赖 temporal buffering，要么依赖稀疏且不稳定的 text-image salience，因此很难在真实机器人部署中同时兼顾速度、鲁棒性和上下文完整性。
- 为此，TEAM-VLA 提出 “Expand + Merge” 两阶段压缩策略：先在当前帧中用 similarity seed + context sampling + spatial expansion 重建较完整的前景区域，再在 backbone 中层用 action-guided soft bipartite merging 压缩深层 token。
- headline 结果需要拆开理解：
  - `109 ms -> 72.1 ms` 与 `1.5x+ speedup` 对应 `OpenVLA-OFT + LIBERO` 上的 latency 口径；
  - “maintaining or even surpassing full-model success” 不是在所有任务与 pruning setting 下都同时成立；
  - 更稳的说法是它在 **training-free、buffer-free** 约束下提供了更优的 accuracy-efficiency tradeoff，而不是单一固定倍率的 universal acceleration claim。

## Methodology Index

- **Observation-only token compression**：只使用当前帧，不依赖历史观测或 temporal buffer。
- **Similarity-based seed selection**：从 image token 与 language embedding 的 similarity 中找到稀疏前景线索。
- **Token Expanding**：用 smoothing convolution scan 将稀疏 seed 扩成更连贯的空间前景区域，同时配合受控随机扩展与 context sampling 保留潜在有效 token。
- **Two-stage compression**：前级 pruning + 中层 merging，而不是只在深层做单次压缩。
- **Action-guided soft bipartite merging**：在中间层保留 top-M action-text-responsive token，并将其余 token 做语义对齐式软合并。
- **Buffer-free / training-free deployment framing**：论文反复强调无需 retraining、无需历史帧缓存、无需额外监督。

## 待核点

- `109 ms -> 72.1 ms`、`1.5x+ speedup`、以及 “无平均性能下降/甚至超过 full model” 目前仍是 bundled summary，后续需要拆到具体 benchmark split 与具体比较对象上。
- 论文把 Token Expanding 与 action-guided merging 一起包装成 TEAM-VLA 主方法；后续 evidence 层可能需要区分“前景重建贡献”和“深层合并贡献”。
- 当前最强直接证据主要基于 LIBERO + OpenVLA-OFT；后续需要明确 TEAM-VLA 的跨架构、跨环境外推边界，而不是直接泛化成一般性 VLA compression 结论。
- 若后续把它放进一般 `token compression` 路线，需要保留“核心是 observation-only + buffer-free，而不是时序缓存式 pruning”这一边界。
