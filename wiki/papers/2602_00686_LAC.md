## Claim
- 页面定位：这是一篇 **learnable adaptive visual token caching** 论文；它的核心贡献是把 token caching 从 heuristic selector 改写成 task-aware computation policy，而不是一般的静态缓存工程。
- LAC 的核心命题是：VLA 推理加速不应继续依赖与任务目标脱节的 heuristic caching / pruning 规则，而应该被重写成一个直接对任务成功负责的 learnable computation policy。论文认为，现有 rule-based token caching 方法通常用 attention、saliency 等 proxy 决定 token 是否复用，但 “模型看哪里” 并不等于 “任务真正需要哪里”，这会让加速策略在动态场景下与任务目标错位。
- 为此，LAC 提出一个轻量的学习式 adaptive caching 框架，用两个协作模块共同决定缓存策略：Cached Token Selector 学习 token 级动态显著性，Cache Ratio Predictor 决定整体 token reuse budget。
- headline 数字需要拆开理解：
  - `1.76x` 对应 wall-clock inference speedup；
  - `+1.9` 个百分点对应 `LIBERO` 平均成功率提升；
  - `+5.0` 个百分点对应 real-world 任务上的成功率提升；
  - `SIMPLER` 又是另一组跨模型验证口径，不应和前述三项混成统一实验结果。
- 更稳的主张是：“learned task-aware caching policy 可以同时优化 efficiency 与 success”，但这种收益建立在 learnable selector、global reuse budget 与特定 base model / benchmark 的组合上，而不是对所有 VLA 与环境都固定成立。

## Methodology Index

- **Learnable adaptive caching**：把 token caching 从 heuristic 规则改成 learnable、task-aware 的 computation policy。
- **Cached Token Selector**：学习 token-level dynamic saliency，决定哪些视觉 token 需要重新计算、哪些可以从缓存中复用。
- **Cache Ratio Predictor**：根据全局场景动态决定 cache ratio / reuse budget，而不是使用固定缓存比例。
- **Optical-flow-based motion prior**：用轻量 optical flow 捕捉场景动态，帮助识别真正会影响任务执行的运动区域。
- **Two-stage training**：先用 attention alignment 做初始化，再在冻结的 VLA backbone 上做 joint optimization。
- **Differentiable relaxation for discrete decisions**：通过可微松弛解决 selector 与 ratio predictor 的离散决策训练问题。

## 待核点

- `1.76x`、`+1.9`、`+5.0` 目前仍是 bundled headline，后续需要分别拆到 LIBERO、SIMPLER、real-world 的具体实验设置和 base model 上。
- 论文把 Cached Token Selector 与 Cache Ratio Predictor 共同包装成 LAC 主体；后续 evidence 层可能需要区分“token-level saliency 学习”的贡献与“global reuse budget 预测”的贡献。
- 论文强调 learned policy 比 heuristic 更 task-aware，但其 motion prior 仍依赖 optical flow；后续需要明确这种设计在极端动态视觉条件下的适用边界，而不是直接泛化为所有场景都同样有效。
- 若后续把它并入一般 `token caching` 路线，需要保留“核心是 learnable task-aware caching，而不是规则式 cache reuse”这一边界。
