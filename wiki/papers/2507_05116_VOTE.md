## Claim

- 页面定位：这是一篇 **single-token action representation + trajectory voting inference recipe** 论文；核心贡献是把整个 action chunk 压到 `<ACT>` token 并在推理期做 trajectory ensemble voting，而不是单一 benchmark 页。
- 这篇论文提出 **VOTE**，它认为现有自回归 VLA 效率低的原因有两个：一是动作 token 太多，二是已经预测出的动作没有被充分复用。为同时解决这两个问题，论文提出一套训练和推理结合的 recipe：先把整个 action chunk 压缩成一个特殊的 `<ACT>` token，再对当前与历史预测动作做 **trajectory ensemble voting**，以提升鲁棒性。
- 作者把 VOTE 同时定位成**速度提升方法**与**性能提升方法**。headline 结果包括：相对 OpenVLA 推理可快约 **`39x`**、在边缘设备上达到约 **`46 Hz`** throughput，同时在 LIBERO 和 SimplerEnv 上也报告了更高成功率。
- 需要显式保留的 caveat 是：这些 headline 数字混合了不同硬件平台、不同 benchmark 和不同 comparison set。`39x / 46 Hz` 的效率数字、`>20%` 的 LIBERO 提升、以及 `+7%` 的 SimplerEnv 提升，并不来自同一张统一表格，因此当前更适合把它们视作作者层面的 summary numbers。

## Methodology Index

- single-token action chunk representation (`<ACT>`)
- action-head decoding from `<ACT>` hidden state
- parallel next-action generation
- trajectory ensemble voting
- efficiency-first VLA optimization recipe
