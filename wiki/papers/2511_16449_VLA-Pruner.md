## Claim

VLA-Pruner 的核心主张是：VLM 里的 semantic-only visual token pruning 直接迁移到 VLA 上会失效，因为 VLA 推理本身具有 dual-system 属性，既包含 high-level semantic understanding，也包含 low-level action execution，因此不能只靠 prefill attention 保 token。论文据此提出一个 plug-and-play、training-free 的 VLA token pruning 方法：同时考虑 vision-language prefill attention 与 action decode attention，并利用机器人操作中的 temporal continuity 去估计 prefill 时刻尚不可见的 action-side importance。作者 claim 该方法能在多个 VLA 架构上取得更优的 speed-performance tradeoff，headline 包括最高 `1.99x` speedup、在高 pruning ratio 下仍保持接近原模型的 manipulation quality，甚至在 `50%` pruning ratio 时还能出现性能反升。

显式 caveat：`1.99x`、`87.5% pruning ratio`、`50% ratio 时性能提升` 以及 cross-architecture generalizability 来自不同模型与不同实验设置。论文真正的中心命题是“dual-system-aware token retention 比 semantic-only pruning 更适合 VLA”，而不是单一固定倍率的加速结论。

## Methodology Index

- **Dual-system view of VLA inference**：把 VLA 推理拆成 vision-language prefill 与 action decode 两个视觉需求不同的阶段。
- **Dual-objective importance criterion**：同时使用 prefill attention 表征 semantic relevance，以及经过 temporal smoothing 估计的 action decode attention 表征 execution precision。
- **Temporal continuity exploitation**：利用连续 manipulation 中相邻 timestep 的 attention overlap 来近似当前动作阶段所需的视觉重要性。
- **Dual-level token selection**：先取 semantic/action 两类显著 patch 的 union 保证 relevance，再做 redundancy filtering，避免简单 attention fusion 带来的偏置与冗余。
- **Training-free / plug-and-play acceleration**：无需重训练，可插到多类带 action-to-vision cross-attention 的 VLA 架构中。
- **Cross-architecture applicability**：论文显式在 OpenVLA、OpenVLA-OFT、`π0` 等不同 VLA 形式上验证。

## 待核点

- `1.99x` speedup、`50%` pruning 可能带来性能提升、以及在 `87.5%` pruning 下仍保持鲁棒，这几类数字来自不同 retention ratio / model / benchmark，后续需要逐项拆证据。
- 论文把 dual-system diagnosis、temporal smoothing heuristic、dual-level selection strategy 合并成一个总方法叙述；后续 evidence 层可能需要拆开“诊断成立”与“具体 heuristic 是否最优”。
- 论文强调 plug-and-play generality，但最强直接证据主要集中在带 action-to-vision cross-attention 的几类 VLA；后续需要明确它的适用边界，而不是泛化成所有 VLA pruning 场景。
