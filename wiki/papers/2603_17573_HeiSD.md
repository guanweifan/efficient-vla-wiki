## Claim
- 页面定位：这是一篇 **hybrid speculative decoding for VLA inference** 论文；其核心贡献是把 retrieval-based 与 drafter-based speculative decoding 按运动学特征做混合调度，并修复 retrieval-based SD 在 embodied setting 下的 accept/reject 问题，而不是提出新的 VLA policy architecture。
- 这篇论文要解决的是：Speculative Decoding 已被视为提升 VLA 推理速度的有前景路线，但现有方法通常只采用 drafter-based 或 retrieval-based 两类 SD 之一，既没有分析两者在 VLA 中的互补性，也没有解决 retrieval-based SD 的 draft rejection、persistent errors，以及 hybrid 使用时 boundary 难以判定的问题。
- 核心主张是：针对 VLA 的多步推理轨迹，不同阶段适合不同类型的 speculative decoding；如果把 retrieval-based SD 与 drafter-based SD 按轨迹运动学特征进行混合使用，并补上 retrieval 侧的 verify-skip 与 sequence-wise relaxed acceptance，就能同时保留两类 SD 的优势，在较高 success rate 下获得更好的推理加速。
- 作者提出 `HeiSD`，其核心包括 retrieval-based SD 的两项优化（`adaptive verify-skip`、`sequence-wise relaxed acceptance`）以及一个用于自动切换 hybrid boundary 的 `kinematic-based fused metric`。
- headline 数字需要拆开理解：
  - 在 simulation benchmark 上，作者报告最高约 `2.45×` speedup，并强调相比纯 drafter-based / pure retrieval-based SD 与 `SpecVLA` 有更强的 speed-success tradeoff。
  - 在 real-world 场景中，作者报告约 `2.06×∼2.41×` speedup，同时保留较高任务成功率，但这对应的是特定 fine-tuned model 与特定 tabletop task setting。
- 更稳的主张是：`HeiSD` 证明 retrieval-based SD 与 drafter-based SD 在 VLA 中具有互补性，且 retrieval-side accept/reject 机制若不做专门修复，就难以稳定进入 embodied deployment。来源：[[raw/2603_17573_HeiSD.pdf]]，第 1-3 页摘要、引言与 Fig. 1；第 6-8 页 Table 3、Table 4 与相关正文。

## 待核点
- `2.45×`、`2.06×∼2.41×` 与 success rate 结果分别来自不同 benchmark 与部署设置，后续不能压成一个无条件的统一提速结论。
- 这篇强调的是 runtime decoding framework，不是新的 VLA backbone / policy；后续 taxonomy 需要避免把它误写成 policy architecture 创新。
- 论文同时使用 simulation `LIBERO` 与 real-world tabletop tasks 验证，但 real-world 结果是基于特定 fine-tuned model 与任务集合，不能泛化成所有 VLA/任务上都能达到相同 speedup。
- `HeiSD` 的性能依赖 retrieval database 质量、verify-skip 策略与 fused metric 阈值；后续若写成通用结论，需要保留这些系统依赖条件。
