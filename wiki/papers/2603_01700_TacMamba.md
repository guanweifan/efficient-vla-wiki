## Claim
- 页面定位：这是一篇 **tactile-visual dual-system + tactile history compression** 论文；它的核心贡献是把高频触觉记忆从低频视觉 VLA 中解耦出来，而不是单纯给现有 VLA 增加一个新模态输入。
- 这篇论文要解决的是：在按钮点击、隐式状态切换、柔性物体接触等视觉模糊 manipulation 中，真正的 ground truth 常常来自高频 tactile feedback，而当前 VLA 多数只能在低频视觉控制回路中同步处理触觉快照，无法持续保留长时 tactile history，因此既难以实时闭环，也难以在长序列中维持物理状态记忆。
- 核心主张是：如果把触觉流与视觉 VLA 解耦成一个“快触觉 / 慢视觉”的双系统结构，用一个 Mamba-based tactile encoder 在 `100Hz` 持续压缩触觉历史，再把压缩状态按需注入低频 VLA planner，就能同时满足硬实时约束与长时 tactile memory 需求。
- 作者提出 `TacMamba`，包括一个 `Mamba-based Tactile History Compressor` 和 `Tactile-Guided Dual-Stage Training`。
- headline 数字需要拆开理解：
  - `88.89%` 准确率与 `0.45 ms` inference latency 描述的是触觉编码器本身的离线分类 / 推理效率；
  - `100%` success 主要对应 sequential button pressing；
  - `95%` 对应 fries packing；
  - `100Hz` 描述的是快触觉控制回路的硬实时要求，不等于整套视觉+触觉系统的统一端到端 VLA latency。来源：[[raw/2603_01700_TacMamba.pdf]]，第 1-2 页摘要与 Fig. 1；第 4-6 页 Table I、Table II、Fig. 4-6。
- 更稳的主张是：`TacMamba` 证明在 tactile-rich manipulation 中，把高频触觉记忆压缩并解耦为独立快回路，比让低频视觉 VLA 同步承担触觉历史更适合满足实时性与长时状态记忆需求。

## 待核点
- `100% success` 主要来自特定物理任务（尤其 sequential button pressing）与特定训练步数下的结果，不应直接泛化为所有 tactile-rich manipulation 的统一水平。
- `0.45 ms` 是 tactile encoder 本身的 inference latency，不等于整套视觉+触觉系统的端到端 VLA latency；后续若写效率结论，需要明确这是 System 1 编码器口径。
- 论文的 system baseline 主要是视觉-only `π0.5`；虽然表中列出其他视觉 memory baseline 的已发表数字，但当前工作更像是在证明“高频 tactile memory”这条路径，而不是全面 benchmark 所有 tactile-VLA。
- `TacMamba` 同时是 hardware-software co-design、触觉编码器论文、以及 VLA 融合论文；后续 taxonomy 需要决定主归类。
