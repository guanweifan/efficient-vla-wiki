## Claim
- 页面定位：这是一篇 **performance-driven differentiable visual token pruning** 论文；核心贡献是用 dynamic query + Gumbel-softmax 做可微分、自适应 token selection，而不是 training-free pruning。
- 这篇论文提出 **LightVLA**，目标是让 VLA 的 visual token pruning 从固定比例、启发式压缩，转向**performance-driven、differentiable、adaptive** 的动态选择。作者的核心判断是：在 VLA 中，视觉 token 是否保留，应该由任务表现驱动，而不是由预先设定的 pruning ratio 或 “magic numbers” 决定。来源：[[raw/2509_12594_LightVLA.pdf]]，第 1 页 Abstract；第 2-4 页 Introduction、Method。
- 方法上的关键主张是：LightVLA 用 **dynamic query** 评估 visual token importance，并通过 **Gumbel-softmax** 实现 differentiable token selection；主版本强调 **不引入额外可训练参数**、不需要额外 auxiliary loss，也不依赖固定保留 token 数。来源：[[raw/2509_12594_LightVLA.pdf]]，第 1 页 Abstract；第 3-4 页 Fig. 2、Sec. III。
- 论文保留了明确 headline numeric claims：相对 **OpenVLA-OFT**，LightVLA 在 **LIBERO** 上实现约 **59.1% FLOPs reduction**、**38.2% latency reduction**，同时平均 success rate 提升约 **2.6%**。这些数字主要来自效率对比表，而 `LightVLA` 在四个 LIBERO suite 上的具体 success-rate 走势则应回到主 benchmark 表单独理解。来源：[[raw/2509_12594_LightVLA.pdf]]，第 1 页 Abstract；第 5 页 Table I、Table II。
- 论文还引入了带额外可训练参数的 **LightVLA\*** 变体，用于探索 learnable query 的上界；但主线仍然是参数无增量、兼容现代 inference framework 的 `LightVLA` 本体。因此这篇更像一篇**可微分 visual token pruning 框架论文**，而不是简单的单次 token pruning trick。来源：[[raw/2509_12594_LightVLA.pdf]]，第 1 页 Abstract；第 6-7 页 Sec. V、Table V、Fig. 4-5。

## 待核点
- `59.1% FLOPs reduction`、`38.2% latency reduction`、`2.6% success-rate improvement` 目前都保留为 headline；后续需要精确拆清它们对应的比较对象、GPU / runtime setting 和是否全部相对 OpenVLA-OFT。
- 主版本 `LightVLA` 与带额外参数的 `LightVLA*` 容易混淆；后续需要在主题页和 evidence 页统一“主方法”和“变体”的叙述口径。
- 论文多次使用 `state-of-the-art`、`best performance` 一类表述，但实验主要集中在 **LIBERO** 与特定 foundation model；后续需要决定在单篇主 claim 中保留多强的 performance 语气。
- LightVLA 强调“no additional trainable parameters”，但仍依赖 fine-tuning 学习 token pruning 行为；后续如果涉及 taxonomy，需要明确它属于 training-based pruning，而非 training-free acceleration。
